Operating Systems Terminology

Term

Definition
operating system The software that controls the overall oeration of a computer
job queue These are jobs residing in mass storage waiting for execution
FIFO (First-in,First-out)
interactive processing This is what allows computer programs to carry on a dialogue with the user
time sharing This was invented to allow multiple users to use the same computer back in the 1970s
multiprogramming This divides time into intervals and only allows the execution of each job in one interval at a time
application software This consists of the programs for performing tasks particular to a machine's utilization
utility software This is software that is fundamental to the computer itself but is not included in the operating system
graphical user interface (GUI) This is what allows the user to move objects around on the computer
shell This is the old GUI which only allowed the user to communicate with the computer by typing in text
kernel This is the internal part of the operating system
device driver These are software units that communicate with the controllers
file manager This coordinates the use of the computer's mass storage facilities
memory manager This coordinates the computer's use of memory
virtual memory This is memory creating by paging between the main storage and the existing memory to create extra memory
boot strapping This transfers the operating system from mass storage into main memory
ROM This is Read Only Memory (its contents can be read but not altered)
deadlock This is when two or more processes are halted because they are waiting for a resource that is allocated to another
spooling This is a technique that holds data for output at a later, but more convenient time
super user (or administrator) This is the person who has highly priveleged access to the operating system
multi-factor authentification This is basically a way to have more than one "password" for a user's acount and this helps the acccount be more secure
